<h1><a id="user-content-系统要求" class="anchor" aria-hidden="true" href="#系统要求"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>系统要求</h1>
<blockquote>
<ul>
<li>Mac must be a 2010 or newer model, with Intel’s hardware support for <strong>memory management unit (MMU) virtualization</strong>; i.e., <strong>Extended Page Tables (EPT)</strong></li>
<li>OS X <strong>10.10.3</strong> Yosemite or newer</li>
<li>At least <strong>4GB of RAM</strong></li>
<li>VirtualBox prior to version 4.3.30 must NOT be installed (it is incompatible with Docker for Mac)</li>
</ul>
</blockquote>
<h1><a id="user-content-安装" class="anchor" aria-hidden="true" href="#安装"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>安装</h1>
<p>两种方式：</p>
<ul>
<li>通过 <code>brew cask install docker</code> 进行安装（推荐）；</li>
<li>登陆到 <a href="https://www.docker.com/docker-mac" rel="nofollow">Docker for Mac</a> 官网下载 Docker.dmg 后安装；</li>
</ul>
<h2><a id="user-content-构成-docker-for-mac-的组件" class="anchor" aria-hidden="true" href="#构成-docker-for-mac-的组件"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>构成 Docker for Mac 的组件</h2>
<p>Docker for Mac 安装后，将得到如下组件</p>
<ul>
<li>Docker Engine</li>
<li>Docker CLI client</li>
<li>Docker Compose</li>
<li>Docker Machine</li>
</ul>
<p>安装后确认</p>
<div class="highlight highlight-source-shell"><pre>docker --version
docker-compose --version
docker-machine --version
docker ps</pre></div>

<h2><a id="user-content-docker-toolbox" class="anchor" aria-hidden="true" href="#docker-toolbox"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><a href="https://docs.docker.com/toolbox/overview/" rel="nofollow">Docker Toolbox</a></h2>
<p>Docker Toolbox 是一种 installer ，用于在不满足 Docker for Mac 和 Docker for Windows apps 运行要求的、older Mac 和 Windows 系统上，快速安装和启动一套 Docker 环境的工具；</p>

<ul>
<li><strong>Docker Machine</strong> for running <code>docker-machine</code> commands</li>
<li><strong>Docker Engine</strong> for running the <code>docker</code> commands</li>
<li><strong>Docker Compose</strong> for running the <code>docker-compose</code> commands</li>

<li>a <strong>shell</strong> preconfigured for a Docker command-line environment</li>
<li>Oracle <strong>VirtualBox</strong></li>
</ul>

<p>Docker Toolbox 使用的是 Oracle <strong>Virtual Box</strong>，而不是 <strong>HyperKit</strong>；因此允许不满足上面的系统要求；</p>
</blockquote>
<h2><a id="user-content-docker-machine" class="anchor" aria-hidden="true" href="#docker-machine"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><a href="https://docs.docker.com/machine/overview/" rel="nofollow">Docker Machine</a></h2>
<p>Docker Machine 的用处：</p>
<ul>
<li>可以用于在 Mac 或 Windows 上安装和运行 Docker</li>
<li>可以 Provision 和 manage 多个远端 Docker hosts</li>
<li>可以 Provision Swarm clusters</li>
</ul>
<p>Docker Machine 是一种工具，允许你在 virtual hosts 上安装 Docker Engine，允许你通过  <code>docker-machine</code> 命令管理这些 hosts ；可以使用 Machine 在你本地 Mac 或 Windows box 中，在你的公司网络中，在你的数据中心中，以及在类似 AWS 或 Digital Ocean 的云提供商中创建 Docker hosts ；</p>
<p>通过 <code>docker-machine</code> 命令你可以 start, inspect, stop 以及 restart 一个受管控的 host ，升级指定的 Docker client 和 daemon ，以及配置指定的 Docker client 与你的 host 进行通信；</p>
<p>将 Machine 的 CLI 指向一处于运行状态的，受管控的 host 后，你就可以直接在那个 host 上运行 docker 命令了；例如，运行 <code>docker-machine env default</code> 以指向一个名为 default 的 host ，并按照屏幕上提示的信息完成 env 的设置，之后就可以运行 <code>docker ps</code> ，<code>docker run hello-world</code> 等指令了；</p>
<p>Machine 是 <strong>Docker v1.12</strong> 出现之前，在 Mac 或 Windows 上运行 Docker 的唯一方式；从 beta program 和 Docker v1.12 开始，Docker for Mac 和 Docker for Windows 已经作为 native apps 可用了，并作为更好的使用方式被推荐；</p>
<h3><a id="user-content-为什么要使用-docker-machine" class="anchor" aria-hidden="true" href="#为什么要使用-docker-machine"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>为什么要使用 Docker Machine</h3>
<p>Docker Machine 令你能够对运行不同 Linux 版本的众多远端 Docker hosts 进行 provision ；<br>
Docker Machine 允许你将 Docker 运行在 older Mac 或 Windows 系统上；</p>
<p>Docker Machine 在如下两种场景下被广泛使用：</p>
<ul>
<li>I have an older desktop system and want to run Docker on Mac or Windows</li>
</ul>
<blockquote>
<p>If you work primarily on an older Mac or Windows laptop or desktop that doesn’t meet the requirements for the new Docker for Mac and Docker for Windows apps, then you need Docker Machine in order to “run Docker” (that is, Docker Engine) locally. Installing Docker Machine on a Mac or Windows box with the Docker Toolbox installer provisions a local virtual machine with Docker Engine, gives you the ability to connect it, and run docker commands.</p>
</blockquote>
<ul>
<li>I want to provision Docker hosts on remote systems</li>
</ul>
<blockquote>
<p>Docker Engine runs natively on Linux systems. If you have a Linux box as your primary system, and want to run docker commands, all you need to do is download and install Docker Engine. However, if you want an efficient way to provision multiple Docker hosts on a network, in the cloud or even locally, you need Docker Machine.</p>
<p>Whether your primary system is Mac, Windows, or Linux, you can install Docker Machine on it and use docker-machine commands to provision and manage large numbers of Docker hosts. It automatically creates hosts, installs Docker Engine on them, then configures the docker clients. Each managed host (”machine”) is the combination of a Docker host and a configured client.</p>
</blockquote>
<h2><a id="user-content-docker-engine" class="anchor" aria-hidden="true" href="#docker-engine"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><a href="https://docs.docker.com/engine/understanding-docker/#/what-is-docker-engine" rel="nofollow">Docker Engine</a></h2>
<p>Docker Engine 属于 client-server 应用模型，主要由以下组件构成：</p>
<ul>
<li>A <strong>server</strong> which is a type of long-running program called a daemon process.</li>
<li>A <strong>REST API</strong> which specifies interfaces that programs can use to talk to the daemon and instruct it what to do.</li>
<li>A command line interface (CLI) <strong>client</strong>  that talks to the daemon (through the REST API wrapper).</li>
</ul>
<p>CLI 使用 Docker 提供的 REST API 来控制 Docker daemon 或与 Docker daemon 进行交互（通过脚本或直接 CLI 命令调用）<br>
许多其他 Docker 应用同样使用底层 API 和 CLI ；</p>
<p>daemon 负责创建和管理各种 Docker 对象，如 <strong>images</strong>, <strong>containers</strong>, <strong>networks</strong> 和 <strong>data volumes</strong> ；</p>
<h2><a id="user-content-docker-engine-vs-docker-machine" class="anchor" aria-hidden="true" href="#docker-engine-vs-docker-machine"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><a href="https://docs.docker.com/machine/overview/#/what-s-the-difference-between-docker-engine-and-docker-machine" rel="nofollow">Docker Engine v.s. Docker Machine</a></h2>
<p>当人们谈起 “Docker” 时，他们通常指的是 Docker Engine， 即由 Docker daemon, REST API 和 CLI 构成的 client-server 应用；Docker Engine 接受来自 CLI 的 docker 命令，例如 <code>docker run &lt;image&gt;</code>, <code>docker ps</code> 和 <code>docker images</code> 等；</p>
<p><a target="_blank" rel="noopener noreferrer" href="https://camo.githubusercontent.com/7845b71d3723ea405e73bcb8fddb68894353ccf94d6374b1ab28f0ba5f875ffb/68747470733a2f2f646f63732e646f636b65722e636f6d2f6d616368696e652f696d672f656e67696e652e706e67"><img src="https://camo.githubusercontent.com/7845b71d3723ea405e73bcb8fddb68894353ccf94d6374b1ab28f0ba5f875ffb/68747470733a2f2f646f63732e646f636b65722e636f6d2f6d616368696e652f696d672f656e67696e652e706e67" alt="Docker Engine" data-canonical-src="https://docs.docker.com/machine/img/engine.png" style="max-width:100%;"></a></p>
<p>Docker Machine 是一种可以 provisioning 和 managing 你的 Dockerized hosts（运行了 Docker Engine 的 hosts）的工具；典型情况下，你会安装 Docker Machine 在你的本地系统上；Docker Machine 具有其自己的命令行客户端 <code>docker-machine</code> 和 Docker Engine 客户端 <code>docker</code> ；你可以使用 Machine 来安装 Docker Engine 到一个或多个虚拟系统中；这些虚拟系统可能是本地的（例如当你在 Mac 或 Windows 上的 VirtualBox 中基于 Machine 安装并运行 Docker Engine 时）或者是远端的（例如当你使用 Machine 在云提供商中对 Dockerized hosts 进行 provision 时）；在有些场景中，Dockerized hosts 会被看作受管理的 “machines” ；</p>

HyperKit 被设计成可以同更高层组件进行交互，例如 <a href="https://github.com/docker/vpnkit">VPNKit</a> 和 <a href="https://github.com/docker/datakit">DataKit</a> ；<br>
HyperKit 当前仅支持采用了 Hypervisor.framework 框架的 Mac OS X ；<br>
HyperKit 是 Docker For Mac 的核心组件；</p>
